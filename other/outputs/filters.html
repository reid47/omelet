Filters

In Omelet, you can use filters to change the way text appears when compiled. Filters can also be used for performing simple operations on numbers and lists. Filters always take the form | filterName and are written at the end of a tag, interpolation, or parenthetical. Multiple filters can be "piped" together on the same text, like (some text | filter1 | filter2). The filters are applied in order, from left to right. 
(div.loud :: omelet is cool | uppercase | append "!!!")
<div class="loud">OMELET IS COOL!!!</div>

Filters with parameters

Every filter takes one implicit parameter: the text to which it is being applied. Some filters take additional parameters, which are listed following the filter name, like so: 
(h1 :: this is text | append " with something appended")
(h1 :: this is text | remove "text")
(h1 :: this is text | replace "text" "cool text")
(h1 :: this is text | truncate 7)
<h1>this is text with something appended</h1>
<h1>this is </h1>
<h1>this is cool text</h1>
<h1>this is...</h1>

Filters on tags

All of the examples above show filters on tags. When a filter is applied to a tag, everything within that tag, between the :: and the |, will be filtered.

Filters on interpolations

Filters can also be applied to interpolations, which are surrounded by curly braces: { and }. Interpolations are how you refer to previously-defined variables and macros. 
def myName = "reid"
(p :: My name is {myName | append "mitchell" | capitalize}.)
<p>My name is Reid Mitchell.</p>

Filters on parentheticals

Sometimes, you'll want to apply filters to text that isn't contained within a tag and isn't an interpolated variable or macro. In this case, you can use a parenthetical. This is just surrounding one piece of the text in parentheses and applying filters to that section. 
(p :: Most of this text is not filtered, but (this part | uppercase) is!)
<p>Most of this text is not filtered, but THIS PART is!</p>
