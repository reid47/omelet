Tags

In Omelet, tags look like this: 
(h1 :: Hello, world!)
The above example compiles into this HTML: 
<h1>Hello, world!</h1>
Just like in HTML, tags can be nested within each other: 
(div ::
    (p :: This paragraph element has (b :: bold) text.)
    (p :: This is a second paragraph.))
<div>
    <p>This paragraph element has <b>bold</b> text.</p>
    <p>This is a second paragraph.</p>
</div>

Attributes

Attributes look much like HTML attributes. They must come after the tag name and before the two colons. 
(div class="box" ::
    (h1 id="headline" :: This is a headline)
    (p :: This text has a (a href="http://example.com" :: link).))
<div class="box">
    <h1 id="headline">This is a headline</h1>
    <p>This text has a <a href="http://example.com">link</a>.</p>
</div>

Special Attributes: class & id

Since class and id attributes are used so often, Omelet lets you use a shorthand, CSS-like syntax for these. Note that only one id attribute is allowed. 
(div#box47.box.special ::
    This is a box.)
<div id="box47" class="box special">
    This is a box.
</div>

Self-closing Tags

All tags listed as void tags in the HTML5 spec are self- closing. These tags are written and compiled as follows. Note that they must stil have the :: at the end. 
(hr ::)
(img src="picture.jpg" ::)
(link rel="stylesheet" type="text/css" href="main.css" ::)
<hr />
<img src="picture.jpg" />
<link rel="stylesheet" type="text/css" href="main.css" />
